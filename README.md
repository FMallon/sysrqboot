# sysrqboot
Perform a Magic SysRq reboot if hung processes or stuck NTFS operations prevent a normal reboot.

This is not intended for public use - it just reboots the system....

I used it once, it worked.... and hopefully it will work the next time so I dont have to keep googling the order of the /proc/sysrq-trigger thing.

It is public because I may need to use it on other systems.

It just reboots the system, that is all....
